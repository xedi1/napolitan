# Usability Test Guide

## هدف
تست قابلیت استفاده با گارسون‌های واقعی قبل از هر Release برای اطمینان از تجربه کاربری روان.

---

## قبل از تست

### آماده‌سازی محیط
- [ ] دستگاه تست آماده (تبلت یا موبایل)
- [ ] دسترسی به اپلیکیشن نصب شده
- [ ] حساب‌های تست ایجاد شده (PIN: 1234, 5678, 0000)
- [ ] سناریوهای تست آماده

### شرکت‌کنندگان
- حداقل 3 گارسون با تجربه متفاوت
- ترجیحاً افرادی که قبلاً با سیستم‌های مشابه کار نکرده‌اند

---

## سناریوهای تست

### سناریو 1: ورود به سیستم
**هدف:** بررسی فرآیند ورود

**مراحل:**
1. اپلیکیشن را باز کنید
2. PIN اشتباه وارد کنید (مثلاً 1111)
3. PIN درست وارد کنید (0000)
4. مشاهده کنید که صفحه ورود بسته می‌شود

**سوالات ارزیابی:**
- آیا فرآیند ورود واضح بود؟
- آیا پیام خطا برای PIN اشتباه مناسب بود؟
- آیا نام کاربر بعد از ورود نمایش داده شد؟

**معیار موفقیت:** ✓ اگر کاربر بدون کمک بتواند وارد شود

---

### سناریو 2: مشاهده وضعیت میزها
**هدف:** بررسی نمایش وضعیت میزها

**مراحل:**
1. وارد سیستم شوید
2. به صحنه 3D نگاه کنید
3. تعداد میزهای خالی و اشغال را بشمارید
4. نوار وضعیت بالای صفحه را بررسی کنید

**سوالات ارزیابی:**
- آیا رنگ‌های وضعیت واضح بودند؟
- آیا شمارش در نوار وضعیت صحیح بود؟
- آیا می‌توانستید میزهای مختلف را تشخیص دهید؟

**معیار موفقیت:** ✓ اگر کاربر بتواند وضعیت را در کمتر از 5 ثانیه تشخیص دهد

---

### سناریو 3: انتخاب میز و تغییر وضعیت
**هدف:** بررسی تعامل با میزها

**مراحل:**
1. روی یک میز کلیک کنید
2. پنل اطلاعات را مشاهده کنید
3. وضعیت را به "اشغال" تغییر دهید
4. وضعیت را به "خالی" برگردانید

**سوالات ارزیابی:**
- آیا کلیک روی میز به خوبی کار می‌کرد؟
- آیا پنل اطلاعات واضح بود؟
- آیا دکمه‌های تغییر وضعیت قابل فهم بودند؟

**معیار موفقیت:** ✓ اگر کاربر بتواند بدون کمک وضعیت را تغییر دهد

---

### سناریو 4: ثبت سفارش
**هدف:** بررسی فرآیند سفارش‌گیری

**مراحل:**
1. یک میز خالی را انتخاب کنید
2. دکمه "سفارش جدید" را بزنید
3. دکمه "منو" را بزنید
4. یک آیتم از منو انتخاب کنید
5. منو را ببندید

**سوالات ارزیابی:**
- آیا دکمه‌ها به اندازه کافی بزرگ بودند؟
- آیا منو خوانا بود؟
- آیا قیمت‌ها واضح بود؟

**معیار موفقیت:** ✓ اگر سفارش در کمتر از 30 ثانیه ثبت شود

---

### سناریو 5: حالت متنی (دسترس‌پذیری)
**هدف:** بررسی حالت جایگزین

**مراحل:**
1. کلید Alt+T را بزنید
2. حالت متنی را مشاهده کنید
3. با کلیدهای جهت‌نما ناوبری کنید
4. Alt+T را دوباره بزنید تا برگردید

**سوالات ارزیابی:**
- آیا حالت متنی قابل استفاده بود؟
- آیا کلید میانبر کار می‌کرد؟
- آیا ناوبری با کیبورد روان بود؟

**معیار موفقیت:** ✓ اگر کاربر بتواند بین حالت‌ها جابجا شود

## چک‌لیست قبل از Release

### عملکردی
- [ ] همه سناریوها با موفقیت اجرا شدند
- [ ] خطایی مشاهده نشد
- [ ] زمان بارگذاری قابل قبول است

### UX
- [ ] حداقل 3 کاربر تست کرده‌اند
- [ ] امتیاز رضایت ≥ 4 از 5
- [ ] مشکلات بحرانی رفع شده‌اند

### Accessibility
- [ ] حالت متنی کار می‌کند
- [ ] کنتراست رنگ‌ها مناسب است
- [ ] فونت خوانا است

---

## اجرای منظم

### قبل از هر Release
1. تست‌های بالا را اجرا کنید
2. فرم ارزیابی را پر کنید
3. مشکلات را در GitHub Issue ثبت کنید
4. مشکلات بحرانی را رفع کنید
5. تست را تکرار کنید

### زمان‌بندی پیشنهادی
- Alpha: 1 نفر، 30 دقیقه
- Beta: 3 نفر، 1 ساعت
- RC: 5 نفر، 2 ساعت
